# خطة تنفيذ مشروع تطبيق "محفظتي الذكية" (Smart Expense Tracker)

تطبيق متكامل لإدارة المصاريف اليومية وتحديد الميزانيات وتتبع الأهداف المالية والادخارية بشكل فوري، مصمم بواجهات عصرية (Premium) ومدمج به نظام إعلانات AdMob متطور ونظام ترقية للنسخة المدفوعة.

---

## 🏗️ الهيكل البرمجي للمشروع (Architecture & Directory Structure)
سنعتمد على هيكل برمجية نظيف ومقسم حسب المميزات (Feature-First) لسهولة الصيانة والتطوير:

```text
lib/
├── core/
│   ├── theme/

│   │   └── currency_helper.dart  # تهيئة وعرض رموز العملات المختلفة
│   └── widgets/
│       ├── ad_banner_widget.dart # ويدجت بنر إعلاني جاهز
│       └── ad_native_widget.dart # ويدجت إعلان مدمج وسط القوائم
├── models/
│   ├── transaction_model.dart    # نموذج بيانات المعاملات (دخل/صرف)
│   ├── category_model.dart       # نموذج بيانات الفئات وألوانها وأيقوناتها
│   ├── budget_model.dart         # نموذج بيانات الميزانيات المحددة
│   └── goal_model.dart           # نموذج بيانات أهداف الادخار
├── providers/
│   ├── transaction_provider.dart # إدارة عمليات الإضافة والتعديل والفلترة
│   ├── budget_provider.dart      # تتبع الميزانيات ونسب الاستهلاك

└── screens/
    ├── dashboard/
    │   ├── dashboard_screen.dart # الشاشة الرئيسية والإحصائيات والرسوم
    │   └── widgets/
    │       ├── balance_card.dart  # بطاقة الرصيد الإجمالي
    │       └── chart_widget.dart  # رسم بياني دائري أو أعمدة للمصاريف
    ├── transaction/
    │   ├── add_transaction_screen.dart # شاشة إضافة معاملة جديدة

        └── settings_screen.dart  # شاشة الإعدادات، الترقية، وتغيير العملة
```

## 📝 تفاصيل خطوات التنفيذ خطوة بخطوة (Detailed Implementation Steps)

### 📂 الخطوة 1: إعداد المشروع وتثبيت الحزم (Setup & Dependencies)
1. إنشاء مشروع Flutter جديد باسم `smart_expense`.
2. تنظيف المشروع الافتراضي وتهيئة ملف `pubspec.yaml` بالحزم التالية:
   * `provider` (إدارة حالة التطبيق).
   * `hive` و `hive_flutter` (قاعدة بيانات محلية سريعة جداً تعمل دون إنترنت).
   * `fl_chart` (رسم التحليلات والرسوم البيانية التفاعلية).
   * `google_mobile_ads` (دمج الإعلانات).
   * `shared_preferences` (حفظ تفضيلات المستخدم مثل اللغة والعملة والوضع الداكن).
   * `intl` (تنسيق الأرقام والتواريخ والعملات).

---

### 🗄️ الخطوة 2: إنشاء النماذج (Models) وتجهيز قاعدة البيانات (Hive)
1. **نموذج المعاملات (TransactionModel):**
   * الحقول: `id`, `title`, `amount`, `date`, `isExpense` (لتحديد هل هي مصروف أم دخل)، `categoryName`.
2. **نموذج الميزانيات (BudgetModel):**
   * الحقول: `id`, `categoryName`, `limitAmount`, `spentAmount`, `monthYear`.
3. **نموذج الفئات (CategoryModel):**
   * الحقول: `name`, `iconCode` (رمز الأيقونة)، `colorValue` (كود اللون المميز).
4. **تهيئة Hive:**
   * تسجيل الـ Adapters وتوليد ملفات `*.g.dart` الخاصة بـ Hive.
   * فتح الصناديق (Boxes) عند إقلاع التطبيق لحفظ وقراءة البيانات فوراً.

---

### 🎨 الخطوة 3: تصميم نظام الألوان والسمات (Theme System)
1. إنشاء `app_theme.dart` لتحديد المظهر المالي الاحترافي:
   * **الوضع الداكن (Dark Mode - الافتراضي فخامة مالية):** خلفيات داكنة فخمة مع درجات اللون الأخضر الزمردي والأحمر المرجاني.
   * **الوضع الفاتح (Light Mode):** ألوان ناصعة ومرتبة مع تباين مريح للعين.
2. برمجة `theme_provider.dart` لحفظ خيار المستخدم وتطبيق الثيم بشكل فوري في التطبيق بأكمله.

---

### 🧠 الخطوة 4: برمجة منطق الحسابات ومزودي الحالة (Providers)
1. **مزوّد المعاملات (TransactionProvider):**
   * دالة `addTransaction()`: إضافة معاملة وحساب الرصيد تلقائياً.
   * دالة `deleteTransaction()`: حذف المعاملة وتعديل الحسابات.
   * دوال الحسابات: حساب إجمالي الدخل لهذا الشهر، إجمالي المصاريف، وصافي الرصيد.
   * فلاتر العرض: عرض معاملات اليوم، الأسبوع، الشهر، أو فرزها حسب فئة محددة.
2. **مزوّد الميزانيات (BudgetProvider):**
   * دالة `setBudgetForCategory()`: وضع ميزانية محددة لفئة معينة.
   * دالة `checkBudgetOverrun()`: فحص ما إذا كان المصروف الجديد يقترب من تجاوز الميزانية المحددة وتحديث حالة شريط التقدم.

---

### 📊 الخطوة 5: بناء واجهات التطبيق الرئيسية (UI Development)

#### 1. لوحة التحكم الرئيسية (Dashboard Screen):
* بطاقة الرصيد العلوي بشكل فخم مع دراسات دخل وصرف الشهر الحالي.
* رسم بياني تفاعلي (Pie Chart) يوضح نسب توزيع المصاريف حسب الفئات (طعام، تسوق، سيارة، إلخ).
* قائمة بآخر 3 معاملات مضافة للوصول السريع إليها.

#### 2. شاشة إضافة معاملة (Add Transaction Sheet):
* حقل كتابة المبلغ بأرقام كبيرة وواضحة.
* أزرار تبديل سريعة بين "مصروف" (أحمر) و "دخل" (أخضر).
* اختيار الفئة عبر شبكة أيقونات (Grid View) ملوّنة وجذابة.
* اختيار التاريخ مع إمكانية إضافة ملاحظة.

#### 3. شاشة السجل والتحليلات (History & Analytics):
* شريط بحث متطور للبحث عن معاملة باسمها.
* فلتر لتحديد الفترة الزمنية (اليوم، الأسبوع الحالي، الشهر الحالي، مخصص).
* فلترة حسب نوع المعاملة (دخل فقط / مصروف فقط) أو حسب فئة معينة.

#### 4. شاشة الميزانيات (Budgets Screen):
* عرض كل فئة وبجوارها الميزانية المحددة لها.
* شريط تقدم يوضح كم تم استهلاكه وكم المتبقي (مثلاً: تم استهلاك 400 من أصل 500 ريال).
* إشارة لونية للتنبيه (أخضر -> أصفر عند الوصول لـ 80% -> أحمر عند التجاوز).

#### 5. شاشة الإعدادات (Settings Screen):
* تبديل العملة الرئيسية (ريال سعودي، دولار، جنيه، درهم، إلخ).
* تفعيل/إيقاف الوضع الداكن.
* خيار ترقية التطبيق وإزالة الإعلانات (Remove Ads) بلمسة واحدة.

---

### 📢 الخطوة 6: ربط ودمج الإعلانات (AdMob Premium Setup)
1. إنشاء ملف `ad_helper.dart` لإدارة معرّفات الوحدات الإعلانية ومراقبة حالة تفعيل النسخة المدفوعة (إذا اشترى المستخدم النسخة المدفوعة، يتم إيقاف الإعلانات تماماً).
2. دمج **إعلانات البنر (Banner Ads):**
   * أسفل لوحة التحكم الرئيسية وشاشة الإعدادات.
3. دمج **إعلانات مدمجة (Native Ads):**
   * ويدجت `AdNativeWidget` بارتفاع 135 بكسل يتم عرضه داخل قائمة سجل المعاملات بعد كل 5 معاملات مضافة، وتصميمه ليتطابق كلياً مع ألوان وبطاقات التطبيق.
4. دمج **الإعلانات البينية (Interstitial Ads):**
   * تظهر فقط عند إتمام خطوة هامة مثل (إضافة ميزانية جديدة، أو الوصول للهدف الادخاري) مع تطبيق كود الحد التكراري لمنع إزعاج المستخدم (مرة كل 5 دقائق كحد أقصى).

---

### 🧪 الخطوة 7: الاختبار النهائي وبناء التطبيق (Testing & Packaging)
1. اختبار تصفح الشاشات وإضافة البيانات بدون إنترنت للتأكد من عمل Hive.
2. اختبار تجاوب الرسوم البيانية مع العمليات الجديدة.
3. فحص الإعلانات وضمان عدم وجود عناصر خارج الحدود (No clipped assets).
4. بناء ملف الـ Release APK النهائي جاهزاً للرفع والاستخدام المباشر.
